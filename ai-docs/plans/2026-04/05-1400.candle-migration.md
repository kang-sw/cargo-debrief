# Candle Migration Plan — ort → candle in embedder.rs

## Context

**Why candle:** ort 2.0.0-rc.12 CoreML EP causes 40GB RSS explosion on first
inference (see ticket `260404-fix-gpu-acceleration`, Phase 1 Result). There is no
upstream fix path. candle 0.10.x is stable on crates.io, has a native
`NomicBertModel` implementation in candle-transformers, and supports Metal
(macOS) + CUDA (Linux/Windows) + CPU fallback without a context-leak bug.

**INDEX_VERSION must be bumped** in `store.rs`. candle produces float32 tensors
through a different inference path than ort; embeddings will differ numerically.
Any existing on-disk index is invalid after this change.

**Public API is unchanged.** `ModelRegistry`, `Embedder::load`, `embed_batch`,
`embed` keep their exact signatures. Only internal implementation changes.

### Researched API surface (candle-transformers 0.10.x)

**NomicBertModel** (`candle_transformers::models::nomic_bert`):
```rust
pub fn load(vb: VarBuilder, config: &Config) -> Result<Self>

pub fn forward(
    &self,
    input_ids: &Tensor,               // dtype: u32, shape: [batch, seq_len]
    token_type_ids: Option<&Tensor>,   // None is fine for nomic-embed
    attention_mask: Option<&Tensor>,   // u32 (0/1), shape: [batch, seq_len]
) -> Result<Tensor>                    // shape: [batch, seq_len, hidden_dim]

// Free functions in the same module:
pub fn mean_pooling(hidden_states: &Tensor, attention_mask: &Tensor) -> Result<Tensor>
pub fn l2_normalize(x: &Tensor) -> Result<Tensor>
```
`mean_pooling` internally casts the mask to `hidden_states.dtype()` — no manual
f32 cast is needed.

**NomicBert Config** (`candle_transformers::models::nomic_bert::Config`, derives
`Deserialize`). The nomic-embed-text-v1.5 `config.json` uses exactly these field
names, so `serde_json::from_str::<Config>` works directly:
```
vocab_size, n_embd (=768), n_head (=12), n_layer (=12), n_inner (=3072),
n_positions (=8192), type_vocab_size, layer_norm_epsilon, rotary_emb_fraction,
rotary_emb_base, rotary_emb_interleaved, qkv_proj_bias, mlp_fc1_bias,
mlp_fc2_bias, activation_function, prenorm, model_type
```

**BertModel** (`candle_transformers::models::bert`):
```rust
pub fn load(vb: VarBuilder, config: &Config) -> Result<Self>

pub fn forward(
    &self,
    input_ids: &Tensor,          // u32
    token_type_ids: &Tensor,     // u32, REQUIRED (not optional)
    attention_mask: Option<&Tensor>,
) -> Result<Tensor>              // shape: [batch, seq_len, hidden_dim]
```
Bert Config uses standard HF naming (`hidden_size`, `num_hidden_layers`, etc.) —
different from NomicBert Config. Must deserialize into separate type.
Use the same `mean_pooling` and `l2_normalize` from nomic_bert module for BertModel
output (they operate on generic Tensors).

**Device API** (`candle_core::Device`):
```rust
Device::new_metal(ordinal: usize) -> Result<Self>   // #[cfg(feature = "metal")]
Device::new_cuda(ordinal: usize) -> Result<Self>    // #[cfg(feature = "cuda")]
Device::Cpu                                          // always available
```

**VarBuilder loading** (`candle_nn::VarBuilder`):
```rust
// Memory-mapped — requires unsafe block:
unsafe { VarBuilder::from_mmaped_safetensors(&[path], DType::F32, &device)? }
```

### HuggingFace repo file layout

| Model | safetensors file | Size | config.json | Notes |
|-------|-----------------|------|-------------|-------|
| nomic-embed-text-v1.5 | `model.safetensors` | ~270 MB | root `config.json` | single file, not sharded |
| bge-large-en-v1.5 | `model.safetensors` | ~1.34 GB | root `config.json` | single file |

### Known pitfalls

- **Token dtype is u32, not i64**: candle embedding layers expect `u32` indices.
  The current ort code uses `i64`. Change all `Vec<i64>` to `Vec<u32>` for both
  token IDs and attention mask when building input tensors.
- **VarBuilder::from_mmaped_safetensors requires unsafe**: wrap in `unsafe { }`.
- **NomicBertModel and BertModel are separate types** with different Config
  structs and different `forward` signatures. Dispatch via an internal enum.
- **Metal/CUDA variants are feature-gated**: wrap `Device::new_metal` and
  `Device::new_cuda` calls in `#[cfg(feature = "metal")]` /
  `#[cfg(feature = "cuda")]` respectively.
- **Mutex may be removable**: candle Tensor and model types are `Send` on CPU;
  Metal/CUDA backends are also thread-safe. Keep the Mutex for now (daemon uses
  a single Embedder; no concurrent access in practice) and document it as
  revisitable.
- **bge-large-en-v1.5 token_type_ids**: BertModel::forward requires a
  non-optional token_type_ids. Always create a zeros Tensor of shape
  `[batch_size, seq_len]` as u32.

## Steps

### 1. Cargo.toml — replace ort with candle

Remove:
```toml
ort = { version = "2.0.0-rc.12", features = ["download-binaries"] }
```

Remove the `[features]` block:
```toml
[features]
gpu = ["ort/coreml"]
cuda = ["ort/cuda"]
```

Add:
```toml
candle-core = { version = "0.10", optional = false }
candle-nn = { version = "0.10" }
candle-transformers = { version = "0.10" }
```

Add new `[features]` block:
```toml
[features]
metal = ["candle-core/metal"]
cuda = ["candle-core/cuda"]
```

Verify exact version available: run `cargo search candle-core` and use the
current 0.10.x on crates.io.

### 2. ModelSpec — replace onnx_path with safetensors fields

Replace the `onnx_path` field with `weights_path` and add `config_path`.
Add a `model_kind: ModelKind` field for dispatch.

```rust
#[derive(Debug, Clone, Copy)]
pub enum ModelKind {
    NomicBert,
    Bert,
}

#[derive(Debug, Clone, Copy)]
pub struct ModelSpec {
    pub name: &'static str,
    pub hf_repo: &'static str,
    pub weights_path: &'static str,   // replaces onnx_path
    pub config_path: &'static str,    // new: path within repo to config.json
    pub tokenizer_path: &'static str,
    pub max_length: usize,
    pub embedding_dim: usize,
    pub model_kind: ModelKind,        // new
}
```

Update `KNOWN_MODELS`:
```rust
static KNOWN_MODELS: &[ModelSpec] = &[
    ModelSpec {
        name: "nomic-embed-text-v1.5",
        hf_repo: "nomic-ai/nomic-embed-text-v1.5",
        weights_path: "model.safetensors",
        config_path: "config.json",
        tokenizer_path: "tokenizer.json",
        max_length: 512,
        embedding_dim: 768,
        model_kind: ModelKind::NomicBert,
    },
    ModelSpec {
        name: "bge-large-en-v1.5",
        hf_repo: "BAAI/bge-large-en-v1.5",
        weights_path: "model.safetensors",
        config_path: "config.json",
        tokenizer_path: "tokenizer.json",
        max_length: 512,
        embedding_dim: 1024,
        model_kind: ModelKind::Bert,
    },
];
```

### 3. ensure_model_files — download safetensors + config.json

Signature stays the same. Replace the local variable `onnx_dest` with
`weights_dest = model_dir.join("model.safetensors")`. Add
`config_dest = model_dir.join("config.json")`.

Return type changes from `(PathBuf, PathBuf)` (onnx, tokenizer) to
`(PathBuf, PathBuf, PathBuf)` (weights, config, tokenizer).

Download `spec.weights_path` (→ `weights_dest`), `spec.config_path`
(→ `config_dest`), and `spec.tokenizer_path` (→ `tokenizer_dest`).
The `download_file` helper is unchanged.

### 4. Embedder struct — replace Session with model enum

Internal private enum (not pub):
```rust
enum EmbedderModel {
    NomicBert(candle_transformers::models::nomic_bert::NomicBertModel),
    Bert(candle_transformers::models::bert::BertModel),
}
```

New `Embedder` struct:
```rust
pub struct Embedder {
    model: Mutex<EmbedderModel>,
    tokenizer: Tokenizer,
    max_length: usize,
    device: Device,      // stored for Tensor construction in embed_batch
}
```

Remove `has_token_type_ids` field — dispatch is now handled by `EmbedderModel`
variant matching.

### 5. Embedder::load — device selection, config, VarBuilder, model construction

```
pub async fn load(model_name: &str, cache_dir: &Path) -> Result<Self>
```

Sequence inside `load`:

**a. Device selection** — wrap each block with appropriate cfg gates:
```rust
#[cfg(feature = "metal")]
let device = Device::new_metal(0).unwrap_or(Device::Cpu);

#[cfg(all(not(feature = "metal"), feature = "cuda"))]
let device = Device::new_cuda(0).unwrap_or(Device::Cpu);

#[cfg(not(any(feature = "metal", feature = "cuda")))]
let device = Device::Cpu;
```

**b. Download files:**
```rust
let (weights_file, config_file, tokenizer_file) =
    ensure_model_files(&spec, cache_dir).await?;
```

**c. Load and deserialize config.json:**
For `ModelKind::NomicBert`:
```rust
let config_str = std::fs::read_to_string(&config_file)?;
let config: candle_transformers::models::nomic_bert::Config =
    serde_json::from_str(&config_str)?;
```
For `ModelKind::Bert`:
```rust
let config_str = std::fs::read_to_string(&config_file)?;
let config: candle_transformers::models::bert::Config =
    serde_json::from_str(&config_str)?;
```
Both deserialization branches go inside a `match spec.model_kind { ... }`.

**d. Load weights via VarBuilder:**
```rust
let vb = unsafe {
    VarBuilder::from_mmaped_safetensors(&[&weights_file], DType::F32, &device)
        .context("failed to load model weights")?
};
```

**e. Construct model:**
```rust
let model = match spec.model_kind {
    ModelKind::NomicBert => {
        let m = NomicBertModel::load(vb, &config)
            .context("failed to build NomicBertModel")?;
        EmbedderModel::NomicBert(m)
    }
    ModelKind::Bert => {
        let m = BertModel::load(vb, &config)
            .context("failed to build BertModel")?;
        EmbedderModel::Bert(m)
    }
};
```

**f. Tokenizer setup:** identical to current — load from `tokenizer_file`,
configure truncation and batch-longest padding.

**g. Return:**
```rust
Ok(Self {
    model: Mutex::new(model),
    tokenizer,
    max_length: spec.max_length,
    device,
})
```

Remove `log_rss` calls and `EMBED_CALL_COUNT` in this step (Phase 4 cleanup).
Executor may defer removal to the cleanup phase and leave them commented out
during the migration to keep diffs small and bisectable.

### 6. embed_batch — candle tensor inference

Tokenization loop is unchanged (same `tokenizers` crate, same truncation/padding
config). The only change is the buffer element type: `Vec<u32>` (not `i64`).

```rust
let mut ids_data = vec![0u32; batch_size * seq_len];
let mut mask_data = vec![0u32; batch_size * seq_len];
```

Build 2D Tensors on device:
```rust
let input_ids = Tensor::from_slice(&ids_data, (batch_size, seq_len), &self.device)
    .context("failed to create input_ids tensor")?;
let attention_mask = Tensor::from_slice(&mask_data, (batch_size, seq_len), &self.device)
    .context("failed to create attention_mask tensor")?;
```

Inference dispatch inside `let mut model = self.model.lock().unwrap();`:
```rust
let hidden_states = match &*model {
    EmbedderModel::NomicBert(m) => m.forward(&input_ids, None, Some(&attention_mask))?,
    EmbedderModel::Bert(m) => {
        let zeros = Tensor::zeros_like(&input_ids)?;
        m.forward(&input_ids, &zeros, Some(&attention_mask))?
    }
};
```

Post-processing (using free functions from nomic_bert module):
```rust
use candle_transformers::models::nomic_bert::{mean_pooling, l2_normalize};

let pooled = mean_pooling(&hidden_states, &attention_mask)?;
let normalized = l2_normalize(&pooled)?;
let embeddings: Vec<Vec<f32>> = normalized.to_vec2::<f32>()?;
```

Return `Ok(embeddings)`.

The `mean_pooling` and `l2_normalize` free functions work on any Tensor regardless
of which model produced the `hidden_states`, so the same post-processing applies
to both NomicBert and Bert outputs.

### 7. store.rs — bump INDEX_VERSION

In `src/store.rs`, increment `INDEX_VERSION` by 1 (current value → +1).
This invalidates all existing on-disk indexes. Users will re-index automatically
on next use (the `ensure_index_fresh` path handles stale version gracefully).

Same increment for `DEPS_INDEX_VERSION` since dep embeddings also change.

### 8. Update unit tests in embedder.rs

The `model_registry_lookup_known` test currently asserts on `onnx_path`. Replace
that assertion with:
```rust
assert_eq!(spec.weights_path, "model.safetensors");
```
Remove any assertion on `onnx_path`.

The integration network tests (`embed_batch_returns_normalized_vectors`,
`embed_batch_truncates_long_input`) require no changes — they test the public
API which is unchanged.

### 9. Remove feature references to old gpu feature (Phase 4)

Remove any `#[cfg(feature = "gpu")]` guards that may remain after the ort
removal. Search for `gpu` in the codebase and remove stale references.
This includes CI / build scripts if they pass `--features gpu`.

## Testing

**Unit (no network):**
- `cargo test` — all 37+ unit tests must pass.
  Key: `model_registry_lookup_known`, `model_registry_lookup_unknown`,
  `model_registry_lookup_bge`. These only test ModelSpec data, not inference.

**Offline integration (no network):**
- `CARGO_DEBRIEF_SKIP_NETWORK=1 cargo test` — all 8 offline integration tests
  must pass. These use mock embedders, so they are unaffected by the inference
  backend change.

**Network integration:**
- `cargo test` (without skip env) — 3 network tests run the real model. Verify:
  - Vectors are 768-dimensional (nomic-embed-text-v1.5)
  - Vectors are L2-normalized (norm within 1e-5 of 1.0)
  - Search quality smoke: test returns expected results

**Manual benchmarking (post-merge):**
- RSS peak on macOS with `--features metal`: target < 2 GB (was 40+ GB with ort CoreML)
- Wall-clock on full ripgrep re-index: baseline is 9m37s on CPU; GPU should be faster
- Search quality eval (24-query suite in `ai-docs/ref/search-quality-eval.md`):
  target >= 15/24 (62.5%, same as ort baseline)

## Success Criteria

1. `cargo build` succeeds (no `--features`), `--features metal`, `--features cuda` each compile.
2. `cargo test` (all layers) passes.
3. Network test: `embed_batch` returns 768-dim L2-normalized vectors for nomic-embed-text-v1.5.
4. `cargo run -- rebuild-index` completes without panic on a real project.
5. `cargo run -- search "query"` returns results.
6. RSS on macOS Metal stays below 2 GB peak during full re-index (verify with `CARGO_DEBRIEF_NO_DAEMON=1`).
7. `INDEX_VERSION` is bumped — stale index triggers clean re-index, not a panic.
