# burn NomicBERT — Manual Implementation Plan

## Context

This is Phase 1 of the GPU acceleration epic (`260405-epic-gpu-embedding-acceleration`):
implement the NomicBERT architecture using burn's `nn` module so the embedder can run
on WGPU (Metal / Vulkan / DX12). The candle implementation remains in place during this
phase; `EmbedderModel` gains a new variant without removing the old ones.

**Why manual, not burn-onnx:** burn-onnx generates build-time Rust from an ONNX graph.
NomicBERT's fused QKV and SwiGLU decompose into many primitive ONNX nodes that may
not preserve the named-parameter structure needed for weight remapping. Manual
implementation gives explicit control over parameter names, which is critical for the
safetensors loading step.

### NomicBERT architecture (from candle-transformers 0.10.2)

Default `config.json` values (`nomic-ai/nomic-embed-text-v1.5`):

| Parameter | Value |
|-----------|-------|
| vocab_size | 30528 |
| n_embd | 768 |
| n_head | 12 |
| n_layer | 12 |
| n_inner | 3072 |
| n_positions | 8192 |
| rotary_emb_fraction | 1.0 (full head_dim rotated) |
| rotary_emb_base | 1000.0 |
| rotary_emb_interleaved | false (non-interleaved / GPT-NeoX style) |
| qkv_proj_bias | false |
| mlp_fc1_bias / mlp_fc2_bias | false |
| prenorm | false (post-norm by default) |

Runtime: read `prenorm` from config.json — the checkpoint value overrides the default.

### HuggingFace weight key paths

```
embeddings.word_embeddings.weight              [30528, 768]
embeddings.token_type_embeddings.weight        [2, 768]
emb_ln.weight                                  [768]   (LayerNorm scale)
emb_ln.bias                                    [768]   (LayerNorm offset)
encoder.layers.{i}.attn.Wqkv.weight           [2304, 768]   (3*n_embd × n_embd)
encoder.layers.{i}.attn.out_proj.weight       [768, 768]
encoder.layers.{i}.mlp.fc11.weight            [3072, 768]
encoder.layers.{i}.mlp.fc12.weight            [3072, 768]
encoder.layers.{i}.mlp.fc2.weight             [768, 3072]
encoder.layers.{i}.norm1.weight               [768]   (LayerNorm scale)
encoder.layers.{i}.norm1.bias                 [768]   (LayerNorm offset)
encoder.layers.{i}.norm2.weight               [768]   (LayerNorm scale)
encoder.layers.{i}.norm2.bias                 [768]   (LayerNorm offset)
```

No bias on Linear projections (`qkv_proj_bias=false`, `mlp_fc1_bias=false`,
`mlp_fc2_bias=false` for the default checkpoint).

### burn nn API surface (v0.20.x)

- `burn::nn::Embedding` — field: `weight: Param<Tensor<B, 2>>`
- `burn::nn::Linear` — fields: `weight: Param<Tensor<B, 2>>`, `bias: Option<Param<Tensor<B, 1>>>`
- `burn::nn::LayerNorm` — fields: `gamma: Param<Tensor<B, 1>>`, `beta: Option<Param<Tensor<B, 1>>>`
- `burn::nn::RotaryEncoding` — `RotaryEncodingConfig { d_model, max_sequence_length, theta }`
- `burn::nn::SwiGlu` — has only 2 Linear layers (`linear_inner`, `linear_outer`); **cannot be
  used directly** for NomicBERT's 3-layer FFN (fc11 value + fc12 gate + fc2 projection)
- Weight loading: `burn_import::safetensors::{SafetensorsFileRecorder, LoadArgs}`

### Weight name mapping challenge

burn derives parameter names from Rust struct field names (dot-separated paths).
Three mismatches between HuggingFace names and burn's conventions:

1. **LayerNorm** — HF uses `.weight` / `.bias`; burn uses `.gamma` / `.beta`
2. **Fused QKV** — HF key path has uppercase `Wqkv`; Rust field must be lowercase `wqkv`
3. **Linear weight transpose** — PyTorch/HF stores Linear weights as `(out, in)`;
   burn's `Linear` stores as `(in, out)`. Use `AdapterType::PyTorch` in `LoadArgs`
   to auto-transpose.

`LoadArgs::with_key_remap(pattern, replacement)` accepts regex + substitution and applies
rules sequentially. Rules to add (applied in order):

```
// 1. LayerNorm weight → gamma
"(emb_ln|encoder\\.layers\\.\\d+\\.norm[12])\\.weight" → "$1.gamma"

// 2. LayerNorm bias → beta
"(emb_ln|encoder\\.layers\\.\\d+\\.norm[12])\\.bias" → "$1.beta"

// 3. Uppercase Wqkv → lowercase wqkv
"(encoder\\.layers\\.\\d+\\.attn)\\.Wqkv\\." → "$1.wqkv."
```

Then set `adapter_type: AdapterType::PyTorch` in `LoadArgs` for weight transposition.

### Step 2 — New file: `src/nomic_bert_burn.rs`

Create `src/nomic_bert_burn.rs`. All burn model code lives here; `embedder.rs` imports
from it. This keeps the file small and the separation clean.

#### 2a. Config struct

Deserializes from the same `config.json` used by candle. Re-use the field names from
`candle_transformers::models::nomic_bert::Config` exactly. Either `use` the candle
config or define a local copy that derives `Deserialize` with the same serde field names:

```rust
#[derive(Debug, Clone, serde::Deserialize)]
#[serde(default)]
pub struct NomicBertConfig {
    pub vocab_size: usize,
    pub n_embd: usize,
    pub n_head: usize,
    pub n_layer: usize,
    pub n_inner: usize,

    pub rotary_emb_fraction: f64,
    pub rotary_emb_base: f64,
    pub rotary_emb_interleaved: bool,
    pub qkv_proj_bias: bool,
    pub mlp_fc1_bias: bool,
    pub mlp_fc2_bias: bool,
    pub prenorm: bool,
}
```

Implement `Default` with the same values as candle's Config::default().
Provide helper methods: `head_dim() -> usize`, `rotary_emb_dim() -> usize`.

Prefer re-using the candle config to avoid drift. If candle stays in Cargo.toml for
this phase (it does), import: `use candle_transformers::models::nomic_bert::Config as NomicBertConfig`.

#### 2b. NomicBertEmbeddings

```rust
#[derive(Module, Debug)]
struct BurnNomicBertEmbeddings<B: Backend> {
    word_embeddings: Embedding<B>,
    token_type_embeddings: Option<Embedding<B>>,
}
```

`forward(input_ids: Tensor<B, 2, Int>, token_type_ids: Option<Tensor<B, 2, Int>>) -> Tensor<B, 3>`

- Word embeddings lookup: `self.word_embeddings.forward(input_ids)` → `[batch, seq, n_embd]`
- Token type: if `token_type_embeddings` is Some, add `tte.forward(zeros)` (zeros tensor of
  same shape as input_ids, dtype Int). Result: `word_emb + tte_emb`.

The `Tensor<B, 2, Int>` type in burn represents a 2D integer tensor (for token indices).

#### 2c. NomicBertSwiGLU (manual, NOT burn::nn::SwiGlu)

burn::nn::SwiGlu has 2 layers but NomicBERT needs 3. Implement manually:

```rust
#[derive(Module, Debug)]
struct BurnNomicBertSwiGLU<B: Backend> {
    fc11: Linear<B>,   // value branch
    fc12: Linear<B>,   // gate branch (SiLU applied)
    fc2: Linear<B>,    // output projection
}
```

`forward(x: Tensor<B, 3>) -> Tensor<B, 3>`:
```
value = fc11.forward(x)
gate  = silu(fc12.forward(x))
fc2.forward(value * gate)
```

Use `burn::tensor::activation::silu(t)` for the SiLU activation (or `t.clone() *
sigmoid(t)` if no dedicated silu function exists in burn 0.20).

#### 2d. NomicBertAttention

```rust
#[derive(Module, Debug)]
struct BurnNomicBertAttention<B: Backend> {
    wqkv: Linear<B>,
    out_proj: Linear<B>,
    rotary_enc: RotaryEncoding<B>,
    num_heads: usize,      // not a parameter — store as plain field
    head_dim: usize,
    n_embd: usize,
}
```

Constructor takes `NomicBertConfig`. Build `RotaryEncoding` with:
```
RotaryEncodingConfig::new(head_dim, n_positions, theta as f32).init(device)
```

`forward(hidden_states: Tensor<B, 3>, attention_mask: Tensor<B, 4>) -> Tensor<B, 3>`:

1. `qkv = wqkv.forward(hidden_states)` → `[batch, seq, 3*n_embd]`
2. Slice into q, k, v each `[batch, seq, n_embd]` using `Tensor::slice`
3. Reshape each to `[batch, seq, num_heads, head_dim]`, then transpose to
   `[batch, num_heads, seq, head_dim]`
4. Apply `rotary_enc.forward(q)` and `rotary_enc.forward(k)` (burn RotaryEncoding accepts
   4D tensors of this shape)
5. `scale = (head_dim as f32).sqrt()`
6. `scores = (q.matmul(k.transpose(-2, -1)) / scale) + attention_mask`
7. `probs = softmax(scores, dim: -1)`
8. `ctx = probs.matmul(v)` → `[batch, num_heads, seq, head_dim]`
9. Transpose back to `[batch, seq, num_heads, head_dim]`, reshape to `[batch, seq, n_embd]`
10. `out_proj.forward(ctx)`

For the attention mask: the caller must pre-expand an `(batch, seq)` 0/1 mask to
`(batch, 1, 1, seq)` with `0.0` for attend and `-1e4` for pad, as a float tensor.
This is computed once per batch in the top-level model forward.

**Note on RotaryEncoding interleaved:** burn's RotaryEncoding implements the non-interleaved
(GPT-NeoX) style by default — this matches NomicBERT's `rotary_emb_interleaved: false`.
Verify at runtime that the embedding output is numerically consistent with candle's
non-interleaved path before removing candle.

#### 2e. NomicBertBlock

```rust
#[derive(Module, Debug)]
struct BurnNomicBertBlock<B: Backend> {
    attn: BurnNomicBertAttention<B>,
    mlp: BurnNomicBertSwiGLU<B>,
    norm1: LayerNorm<B>,
    norm2: LayerNorm<B>,
    prenorm: bool,     // plain bool, not a parameter
}
```

`forward(hidden_states: Tensor<B, 3>, attention_mask: Tensor<B, 4>) -> Tensor<B, 3>`:

Post-norm (prenorm=false, the default):
```
attn_out = attn.forward(hidden_states, attention_mask)
hidden   = norm1.forward(hidden_states + attn_out)
mlp_out  = mlp.forward(hidden)
norm2.forward(hidden + mlp_out)
```

Pre-norm (prenorm=true):
```
normed1  = norm1.forward(hidden_states)
attn_out = attn.forward(normed1, attention_mask)
hidden   = hidden_states + attn_out
normed2  = norm2.forward(hidden)
mlp_out  = mlp.forward(normed2)
hidden   + mlp_out
```

#### 2f. BurnNomicBertModel (top-level, public)

```rust
#[derive(Module, Debug)]
pub struct BurnNomicBertModel<B: Backend> {
    embeddings: BurnNomicBertEmbeddings<B>,
    emb_ln: LayerNorm<B>,
    encoder_layers: Vec<BurnNomicBertBlock<B>>,  // named "encoder.layers" via key remap
}
```

**Weight naming pitfall:** burn's Module derive uses field names as key path segments.
`encoder_layers` would produce keys like `encoder_layers.0.attn.wqkv.weight`, but the
HF checkpoint uses `encoder.layers.0.attn.Wqkv.weight`. A 4th remap rule handles the
Rust field name → HF path translation:

```
// 4. encoder_layers → encoder.layers in key paths
"encoder_layers\\.(\d+)\\." → "encoder.layers.$1."
```

Or alternatively, wrap `Vec<BurnNomicBertBlock<B>>` in a named struct:

```rust
#[derive(Module, Debug)]
struct BurnNomicBertEncoder<B: Backend> {
    layers: Vec<BurnNomicBertBlock<B>>,
}
```

And use `encoder: BurnNomicBertEncoder<B>` as the field name on `BurnNomicBertModel`.
This produces `encoder.layers.{i}.*` naturally — **preferred over key remapping**.

`forward(input_ids: Tensor<B, 2, Int>, attention_mask: Tensor<B, 2, Int>) -> Tensor<B, 3>`:

1. `hidden = embeddings.forward(input_ids, None)`
2. `hidden = emb_ln.forward(hidden)`
3. Compute extended mask: cast attention_mask to float, unsqueeze to `[batch, 1, 1, seq]`,
   then: `mask = mask * 0.0 + (1.0 - mask) * -1e4` (or use where/select)
4. Loop: `for layer in &encoder.layers { hidden = layer.forward(hidden, mask); }`
5. Return `hidden`

#### 2g. Weight loading function

```rust
pub fn load_nomic_bert_burn<B: Backend>(
    weights_path: &Path,
    config: &NomicBertConfig,
    device: &B::Device,
) -> anyhow::Result<BurnNomicBertModel<B>>
```

Body:
```rust
let model = BurnNomicBertModel::init(config, device);
let recorder = SafetensorsFileRecorder::<FullPrecisionSettings>::new();
let load_args = LoadArgs::new(weights_path.to_path_buf())
    .with_key_remap(
        r"(emb_ln|encoder\.layers\.\d+\.norm[12])\.weight",
        "$1.gamma",
    )
    .with_key_remap(
        r"(emb_ln|encoder\.layers\.\d+\.norm[12])\.bias",
        "$1.beta",
    )
    .with_key_remap(
        r"(encoder\.layers\.\d+\.attn)\.Wqkv\.",
        "$1.wqkv.",
    )
    .with_adapter_type(AdapterType::PyTorch);  // transposes Linear weights
let model = model.load_record(recorder.load(load_args, device)?);
Ok(model)
```

`BurnNomicBertModel::init` is a constructor that allocates the module tree with the
right shapes (all `Linear::new` calls use config dims). This lets burn's Module derive
wire up the record loading.

#### 2h. Post-processing free functions (burn tensor ops)

```rust
pub fn burn_mean_pooling<B: Backend>(
    hidden: Tensor<B, 3>,
    attention_mask: Tensor<B, 2, Int>,
) -> Tensor<B, 2>

pub fn burn_l2_normalize<B: Backend>(x: Tensor<B, 2>) -> Tensor<B, 2>
```

`burn_mean_pooling`:
1. `mask_f = attention_mask.float()` — cast to float
2. `mask_exp = mask_f.unsqueeze_dim(2)` → `[batch, seq, 1]`, then broadcast to `[batch, seq, n_embd]`
3. `sum_hidden = (hidden * mask_exp).sum_dim(1)` → `[batch, n_embd]`
4. `sum_mask = mask_f.sum_dim(1).clamp_min(1e-9f32)` → `[batch, 1]`
5. `sum_hidden / sum_mask`

`burn_l2_normalize`:
1. `norm = x.clone().powf(2.0).sum_dim(1).sqrt()` (keepdim)
2. `x / norm`

### Step 5 — Integration test

Add a `#[tokio::test] #[ignore]` test in `tests/integration_network.rs`:

```rust
async fn burn_nomic_bert_produces_normalized_768d_vectors()
```

Loads `nomic-embed-text-v1.5-burn` from the same cache dir, calls `embed_batch` on two
short texts, asserts:
- 2 output vectors
- each has 768 dimensions
- each is L2-normalized (norm within 1e-5 of 1.0)
- cosine similarity between the two vectors is < 0.99 (they differ)

This test is the integration gate before candle can be removed in a later phase.

### Step 6 — store.rs: bump INDEX_VERSION and DEPS_INDEX_VERSION

Bump both version constants by 1. This invalidates on-disk indexes; users re-index
automatically via `ensure_index_fresh`.

## Testing

**Offline (no network):**
- `cargo check` and `cargo check --features wgpu` must pass — no inference needed
- `cargo test` (unit tests only) — model registry lookup tests must pass for the
  new `nomic-embed-text-v1.5-burn` entry

**Network (integration gate):**
- `cargo test burn_nomic_bert_produces_normalized_768d_vectors` (after model download)
- Numerical check: embed the same text with candle path and burn path; cosine similarity
  must be > 0.99 (burn output approximates candle output within float precision)

## Success Criteria

1. `cargo check` and `cargo check --features wgpu` succeed without errors.
2. `cargo test` passes all offline unit tests.
3. Network integration test: burn path produces 768-dim L2-normalized vectors.
4. Numerical consistency: cosine similarity between candle and burn embeddings for the
   same text is ≥ 0.99.
5. No change to `Embedder::load`, `embed_batch`, or `embed` public signatures.
6. `INDEX_VERSION` bumped — stale index triggers clean re-index, not panic.

## Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| burn `RotaryEncoding` uses interleaved RoPE by default | Verify non-interleaved mode in RotaryEncodingConfig; check burn 0.20 source if docs are unclear |
| `SafetensorsFileRecorder` key remap order matters | Apply LayerNorm remap before Wqkv remap; test each rule in isolation with `LoadArgs::debug = true` |
| `AdapterType::PyTorch` may not exist in burn 0.20 | Check `burn_import::safetensors::AdapterType` variants; fallback: transpose weight tensors manually after loading |
| burn's `Module` derive field ordering for Vec<Block> differs from expected | Test with `debug=true` on LoadArgs to print resolved key paths |
| burn tensor data extraction API changed in 0.20 | Check `Tensor::into_data()` → `TensorData` → `.value` chain against docs.rs/burn/0.20 |
| NomicBERT prenorm=true in checkpoint (different from default) | Implement both prenorm branches in BurnNomicBertBlock; config drives dispatch |
