# Embedding Pipeline

## Context

Phase 1B is complete: `Chunk` already carries `embedding: Option<Vec<f32>>` and
`IndexData` already carries `embedding_model: Option<String>`. The scaffolding is in
place; this phase fills in the embedder logic.

Key decisions from research ticket `260403-research-rag-architecture`:
- Inference via `ort` (ONNX Runtime) — Rust-native, no Python dep.
- Default model: `nomic-embed-code` (137M, code-specialized, ONNX export available).
- Models cached in `dirs::data_dir()/debrief/models/` — never per-project.
- Changing model invalidates index; detection is by comparing `IndexData.embedding_model`
  vs current config at indexing time (not in this module — embedder is unaware of the
  store).
- Config layer already has `embedding_model: Option<String>` with full merge semantics.
- `config_paths` and `load_config` exist; only `save_config` is missing.

The `ort` crate v2.x API is used. Note: `ort` v2 changed from v1 (no `Environment`,
`Session::builder()` replaces `SessionBuilder::new(&env)`). Use v2 API throughout.

Pooling strategy: **mean pooling** over `last_hidden_state` followed by **L2
normalization**. This is standard for nomic-embed-code (which uses CLS by default but
mean pooling is also supported and more commonly implemented for BERT-family models in
Rust). Confirm against the model card; if the model card specifies CLS pooling, use
index 0 instead.

No `hf-hub` crate — keep downloads simple with `reqwest` + direct HuggingFace URLs.

## Steps

### 1. Add dependencies to `Cargo.toml`

```toml
[dependencies]
ort = { version = "2", features = ["download-binaries"] }
tokenizers = "0.19"
reqwest = { version = "0.12", features = ["stream"] }
indicatif = "0.17"
ndarray = "0.15"
```

- `ort` with `download-binaries` auto-fetches the ONNX Runtime native library.
- `ndarray` is needed for tensor construction (`ort` v2 accepts `ndarray::Array`).
- `reqwest` with `stream` for chunked download with progress.
- `indicatif` for download progress bar (user-facing; model download can take seconds).

### 2. Create `src/embedder.rs` — public API

Declare `pub mod embedder;` in `src/lib.rs` (append after existing mods).

The module exposes:

```rust
/// Known model identifiers and their HuggingFace repository locations.
pub struct ModelRegistry;

impl ModelRegistry {
    /// Resolve a user-supplied model name to a `ModelSpec`.
    /// Returns `None` for unknown names.
    pub fn lookup(name: &str) -> Option<ModelSpec>;

    /// Name of the default model used when config has no `embedding_model`.
    pub const DEFAULT_MODEL: &'static str = "nomic-embed-code";
}

/// Download coordinates for a model.
#[derive(Debug, Clone)]
pub struct ModelSpec {
    /// Short user-facing name (e.g. "nomic-embed-code").
    pub name: &'static str,
    /// HuggingFace repo ID (e.g. "nomic-ai/nomic-embed-text-v1.5").
    pub hf_repo: &'static str,
    /// Path within the repo to the ONNX model file.
    pub onnx_path: &'static str,
    /// Path within the repo to the tokenizer file.
    pub tokenizer_path: &'static str,
    /// Sequence length cap for tokenization.
    pub max_length: usize,
    /// Output embedding dimension.
    pub embedding_dim: usize,
}

/// Loaded embedder ready for inference.
pub struct Embedder {
    session: ort::Session,
    tokenizer: tokenizers::Tokenizer,
    max_length: usize,
}

impl Embedder {
    /// Load or download the named model into `cache_dir`.
    /// Displays a progress bar if a download is needed.
    pub async fn load(model_name: &str, cache_dir: &Path) -> Result<Self>;

    /// Embed a batch of texts. Returns one `Vec<f32>` per input text.
    /// Texts exceeding `max_length` tokens are truncated.
    pub fn embed_batch(&self, texts: &[&str]) -> Result<Vec<Vec<f32>>>;

    /// Convenience wrapper around `embed_batch` for a single text.
    pub fn embed(&self, text: &str) -> Result<Vec<f32>>;
}
```

Internals (not part of public contract, executor decides implementation):
- `ensure_model_files(spec: &ModelSpec, cache_dir: &Path) -> Result<(PathBuf, PathBuf)>` —
  returns `(onnx_path, tokenizer_path)`. If both files exist, returns immediately. If
  either is missing, downloads all files for the spec with a progress bar. Partial
  downloads are written to `*.tmp` and renamed atomically on completion.
- Tokenization produces `input_ids` (i64), `attention_mask` (i64). `token_type_ids` is
  omitted — nomic-embed-code does not require it (the ONNX model may or may not list it
  as an optional input; skip if not present in session input names).
- Mean pooling: average `last_hidden_state[batch, seq_len, dim]` over `seq_len`, gated by
  `attention_mask` to exclude padding. L2-normalize the result before returning.

### 3. Model registry entries

Seed the registry with two models. Add more without changing the API.

| Name | `hf_repo` | `onnx_path` | `max_length` | `embedding_dim` |
|---|---|---|---|---|
| `nomic-embed-code` | `nomic-ai/nomic-embed-text-v1.5` | `onnx/model.onnx` | 512 | 768 |
| `bge-large-en-v1.5` | `BAAI/bge-large-en-v1.5` | `onnx/model.onnx` | 512 | 1024 |

`tokenizer_path` is `tokenizer.json` for both.

HuggingFace download URL pattern:
```
https://huggingface.co/{hf_repo}/resolve/main/{file_path}
```

### 4. `config.rs` — add `save_config`

Add a single new function:

```rust
/// Write `config` to `path` as TOML, creating parent directories if needed.
/// Only fields present in `Config` are written; `None` fields are omitted.
pub fn save_config(path: &Path, config: &Config) -> Result<()>;
```

Implementation: serialize with `toml::to_string_pretty`, `create_dir_all` on parent, then
`fs::write`. `Config` already derives `Serialize`.

No changes to `Config` struct, `merge`, or any other existing function.

### 5. `service.rs` — implement `set_embedding_model` on `InProcessService`

```rust
async fn set_embedding_model(
    &self,
    project_root: &Path,
    model: &str,
    global: bool,
) -> Result<()>
```

Body:
1. Validate `model` against `ModelRegistry::lookup(model)`. Return `anyhow::bail!` with a
   helpful message if unknown. (Prevents silent typos from polluting config.)
2. Call `config_paths(project_root)`.
3. If `global`:
   - Target path = `paths.global.ok_or_else(...)` (rare: error if no home dir).
   - Load existing global config with `load_config` of just the global layer (or default),
     set `embedding_model = Some(model.to_string())`, save with `save_config`.
4. Else (project scope):
   - Target path = `paths.project.ok_or_else(...)` — error if not in a git repo.
   - Same load-set-save pattern on the project config file.
5. No re-index is triggered here. The model mismatch is detected at index time by comparing
   `IndexData.embedding_model` vs loaded config. The CLI already prints the scope message
   after this method returns (see `main.rs`).

### 6. `store.rs` — no changes

`IndexData` already has all needed fields. `INDEX_VERSION` stays at 1. `Chunk.embedding`
is `Option<Vec<f32>>` already. When the embedding pipeline fills in embeddings during
indexing, the data flows through existing paths.

### 7. `lib.rs` — add `pub mod embedder;`

Append to the existing module list. No other changes.

## Testing

### Unit tests (in `src/embedder.rs`)

- **`model_registry_lookup_known`** — `ModelRegistry::lookup("nomic-embed-code")` returns
  `Some(_)` with correct fields. TDD: write before registry entry.
- **`model_registry_lookup_unknown`** — `ModelRegistry::lookup("not-a-model")` returns
  `None`.
- **`save_config_round_trips_model_name`** (in `src/config.rs`) — write a config with
  `embedding_model = Some("test-model")`, reload with `load_config`, assert field matches.
  TDD: write before `save_config`.
- **`save_config_creates_parent_dirs`** — target path in a nonexistent subdirectory;
  `save_config` succeeds.

### Integration / manual tests

- **`set_embedding_model_project_scope`** — create a temp git dir, call
  `InProcessService::set_embedding_model` with `global: false`, assert `.debrief/config.toml`
  contains the model name and passes `load_config`.
- **`set_embedding_model_global_scope`** — same with `global: true`, assert the global path
  contains the model name. (Requires mocking `dirs::config_dir` or using a temp path;
  simplest approach: expose a test helper or skip in CI.)
- **`embed_batch_returns_normalized_vectors`** (integration, gated behind
  `#[cfg(feature = "integration-tests")]` or `#[ignore]`) — downloads the default model
  (or uses a pre-downloaded copy), embeds two texts, asserts output length equals
  `embedding_dim`, asserts each vector is approximately L2-normalized (`|v| ≈ 1.0`).
- **`embed_batch_truncates_long_input`** — manually verify no panic on input exceeding
  `max_length` tokens.

Gate model download tests behind `#[ignore]` by default so CI does not hit the network.

## Success Criteria

1. `cargo build` succeeds with new dependencies.
2. `cargo test` passes (all non-ignored tests).
3. `cargo run -- set-embedding-model nomic-embed-code` writes the model name to
   `.debrief/config.toml` and prints the scope confirmation.
4. `cargo run -- set-embedding-model --global bge-large-en-v1.5` writes to global config.
5. `cargo run -- set-embedding-model unknown-model` fails with an error mentioning the
   unknown model name.
6. `Embedder::load` can be called in a `#[ignore]`d integration test and produces
   L2-normalized embeddings of the correct dimension.
