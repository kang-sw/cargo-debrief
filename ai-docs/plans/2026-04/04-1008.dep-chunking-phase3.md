# Phase 3: Unified Search + Config

## Context

Ticket: `260404-feat-dependency-chunking`

Phase 2 delivered: `DepsIndexData` serialized to `deps-index.bin`,
`run_deps_index` pipeline, and `ensure_deps_index_fresh` called from
`search` (result discarded with `_deps_data`). Phase 3 wires that data
into live search results, adds origin-based score penalty, CLI `--no-deps`
flag, config exclude list, and dep-aware `overview`.

### Key Decisions

**Merge strategy**: Single unified `SearchIndex` built from combined
`project_chunks + dep_chunks`. No two-index merge or result interleaving.
Pros: the existing HNSW ANN search + boosting pipeline is unchanged; dep
penalty is injected as another score modifier applied inside
`search_by_vector`. Cons: none significant at this scale.

**Origin-based score penalty**: Applied inside `search_by_vector` alongside
symbol-name boosting, using `chunk.origin`. New constant
`DEP_ORIGIN_PENALTY: f64 = 0.1`. Penalty is not configurable â€” same pattern
as `EXACT_SYMBOL_MATCH_BOOST` / `PARTIAL_SYMBOL_MATCH_BOOST`. A dep chunk
with an exact symbol match nets `raw + 0.3 - 0.1 = raw + 0.2`; a project
chunk with the same exact match nets `raw + 0.3`. Ordering is preserved for
equally relevant results.

**`--no-deps` flag**: `bool` flag on `Search` CLI subcommand. Propagated as
`include_deps: bool` parameter added to `DebriefService::search`. When
`false`, dep chunks are excluded from the search pool before
`SearchIndex::build` and `ensure_deps_index_fresh` is not called (skip the
freshness work entirely).

**Config exclude list**: `dependencies.exclude` in `Config`. Applied in
`run_deps_index` after `discover_dependency_packages` returns â€” filter before
chunking + embedding. `discover_dependency_packages` stays pure.

**`SearchResult::origin`**: Add `origin: ChunkOrigin` field. Populated from
`chunk.origin.clone()` in `search_by_vector`. Enables origin-aware display in
`main.rs` without additional lookups.

**Dep overview**: New trait method `dep_overview(project_root, crate_name)`.
Loads dep index, filters chunks by crate name, formats same as project
`overview`. CLI: `file` on `Overview` becomes `Option<PathBuf>`, new
`--dep <name>` flag added.

### Step 3 â€” Add `DEP_ORIGIN_PENALTY` and `apply_dep_penalty` to `src/search.rs`

```rust
const DEP_ORIGIN_PENALTY: f64 = 0.1;

fn apply_dep_penalty(score: f64, origin: &ChunkOrigin) -> f64 {
    match origin {
        ChunkOrigin::Dependency { .. } => score - DEP_ORIGIN_PENALTY,
        ChunkOrigin::Project => score,
    }
}
```


### Step 5 â€” Update `InProcessService::search`

Replace the Phase 2 placeholder `_deps_data` logic with the live merge:

```rust
async fn search(&self, project_root: &Path, query: &str, top_k: usize, include_deps: bool)
    -> Result<Vec<SearchResult>>
{
    let config = load_config(&config_paths(project_root))?;
    let model_name = config.embedding_model.as_deref()
        .unwrap_or(ModelRegistry::DEFAULT_MODEL);
    let embedder = make_embedder(model_name).await?;

    let index_data = ensure_index_fresh(project_root, &embedder).await?;

    // Flatten project chunks.
    let mut flat_chunks: Vec<(PathBuf, Chunk)> = index_data
        .chunks
        .into_iter()
        .flat_map(|(path, chunks)| chunks.into_iter().map(move |c| (path.clone(), c)))
        .collect();

    // Append dep chunks if requested.
    if include_deps {
        let deps_data = ensure_deps_index_fresh(project_root, &embedder).await?;
        for chunk in deps_data.chunks {
            // Key doesn't affect search results â€” origin is on the chunk itself.
            flat_chunks.push((PathBuf::from(&chunk.metadata.file_path), chunk));
        }
    }

    let search_index = SearchIndex::build(flat_chunks)?;
    search_index.search(query, &embedder, top_k)
}
```

### Step 7 â€” Add `dep_overview` to `DebriefService` and implement it

Trait method:

```rust
fn dep_overview(
    &self,
    project_root: &Path,
    crate_name: &str,
) -> impl Future<Output = Result<String>> + Send;
```

`InProcessService` implementation:

1. Load dep index: `store::load_deps_index(&deps_index_path(project_root)?)?`
   â†’ return `Err` if `None` ("no dependency index; run `rebuild-index` first").
2. Filter to chunks where `chunk.origin` matches `ChunkOrigin::Dependency { crate_name: c, .. }` with `c == crate_name`.
3. Filter to `chunk.metadata.chunk_type == ChunkType::Overview`.
4. Sort by visibility (same key as `overview`: Pub â†’ PubCrate â†’ PubSuper â†’ Private).
5. Join `display_text` with `"\n\n"`.
6. If empty: `anyhow::bail!("no overview chunks found for dependency {crate_name:?}")`.

```rust
Command::Search { query, top_k, no_deps } => {
    let results = service.search(&project_root, &query, top_k, !no_deps).await?;
    for (i, r) in results.iter().enumerate() {
        let dep_label = match &r.origin {
            ChunkOrigin::Dependency { crate_name, .. } => {
                format!(" [dep: {crate_name}]")
            }
            ChunkOrigin::Project => String::new(),
        };
        println!(
            "#{} [score: {:.4}]{} {}:{}-{}",
            i + 1, r.score, dep_label, r.file_path, r.line_range.0, r.line_range.1,
        );
        if !r.module_path.is_empty() {
            println!("// in {}", r.module_path);
        }
        println!("{}", r.display_text);
        println!();
    }
}
```

## Testing

**Unit tests** (in the relevant modules, no network/model needed):

**`test_dep_penalty_applied`** (`src/search.rs`) â€” build an index with one
project chunk and one dep chunk at identical embedding direction (same raw
similarity). Query with the same vector. Assert the project chunk ranks first
and its score equals the dep chunk's score plus `DEP_ORIGIN_PENALTY`.

**`test_dep_penalty_does_not_affect_project_chunks`** â€” same as above but
assert project chunk score is unchanged compared to a baseline with no dep
chunks.

**`test_exclude_list_filters_packages`** â€” construct a `Vec<DepPackageInfo>`
with two entries; mock the config exclude list to contain one name; apply the
filter logic; assert only the non-excluded entry remains.

**`test_dep_overview_empty_index_returns_error`** â€” write an empty
`DepsIndexData` to a tempdir; call `dep_overview`; assert `Err` with message
containing "no dependency index" or "no overview chunks".

**Integration tests** (with a pre-built dep index from `#[ignore]` or
self-project):

**`test_search_no_deps_excludes_dep_results`** â€” run `search` with
`include_deps = false`; assert all returned `origin` values are
`ChunkOrigin::Project`.

**`test_search_with_deps_includes_dep_results`** â€” run `search` with
`include_deps = true` on a project with known deps; assert at least one
result has `origin == ChunkOrigin::Dependency { .. }`.

**`test_config_exclude_list_round_trip`** â€” write a config TOML with
`[dependencies]\nexclude = ["syn"]`, load via `load_config`, assert
`config.dependencies.unwrap().exclude == Some(vec!["syn".to_string()])`.

## Success Criteria

- `cargo test` passes with no new failures.
- `cargo clippy` passes with no warnings on new or modified code.
- `cargo debrief search "query"` returns both project and dep results;
  dep results show `[dep: crate_name]` in output.
- `cargo debrief search "query" --no-deps` returns only project results
  (no `[dep:]` labels in output).
- A project config with `[dependencies]\nexclude = ["syn"]` causes `syn`
  chunks to be absent from the dep index after `rebuild-index`.
- `cargo debrief overview --dep serde` prints overview chunks from the serde
  dep index (or a clean error if not indexed).
- `cargo debrief overview src/main.rs` still works as before (no regression).
- All `SearchResult` values in `search` responses have `origin` set to the
  correct variant.
