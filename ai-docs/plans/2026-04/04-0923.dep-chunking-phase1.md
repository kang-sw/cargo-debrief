# Phase 1: Chunk Model + Dependency Discovery

## Context

Ticket: `260404-feat-dependency-chunking`

Phase 1 is model + discovery only. No indexing pipeline changes (Phase 2),
no search changes (Phase 3). After this phase, the codebase has:
- `ChunkOrigin` enum on `Chunk` with serde round-trip
- `INDEX_VERSION` bumped to 4
- A `deps` module with a tested function that returns one `DepPackageInfo`
  per non-workspace dependency, with `root_deps` populated via BFS

### Key Decisions

**cargo_metadata crate vs raw JSON**: Use the `cargo_metadata` crate
(`cargo_metadata = "0.18"`). The resolve graph JSON is deeply nested — parsing
it manually is error-prone. The crate provides typed `Package`, `Resolve`,
and `Node` structs that map directly to the needed fields. The single added
dependency is justified by the reduction in bespoke JSON parsing.

**Module placement**: New `src/deps.rs`. It has no overlap with `git.rs`
(git diff/ls-files) or `service.rs` (orchestration). A single flat file
is sufficient for Phase 1; promote to `src/deps/` only if Phase 2 grows it
past a single clear responsibility.

**BFS strategy**: Forward multi-source BFS. For each direct dependency name
of any workspace member, BFS forward through the resolve graph and add that
name to every reachable package's `root_deps` set. Result:
`HashMap<PackageId, BTreeSet<String>>`. Converting to `Vec<String>` (sorted)
per package is deterministic and testable.

**`src_root` vs `manifest_path`**: Expose `src_root: PathBuf` on
`DepPackageInfo` set to `manifest_path.parent()`. Phase 2 will walk
`src_root/src/` for `.rs` files. Exposing the manifest parent (not `src/`
directly) keeps Phase 1 correct even for the edge case of crates whose
`[lib] path` points outside `src/` — Phase 2 can refine if needed.

**Backward compatibility**: `Chunk.origin` is annotated
`#[serde(default)]`. Old bincode indexes are still silently discarded by
the version bump (3 → 4), but the `Default` impl is useful for tests and
construction sites in `rust.rs`.

pub enum ChunkOrigin {

        crate_name: String,

### Step 5 — Create `src/deps.rs`

Public surface (the executor writes the implementations):

```rust
/// Information about one non-workspace dependency package as returned by
/// `cargo metadata`, enriched with root-dependency reachability.
#[derive(Debug, Clone, PartialEq)]
pub struct DepPackageInfo {
    /// Crate name (matches `package.name` in `cargo metadata`).
    pub crate_name: String,
    /// Semver version string.
    pub crate_version: String,
    /// Directory containing the package's `Cargo.toml`
    /// (i.e. `manifest_path.parent()`).
    /// Phase 2 will look for `.rs` files under `src_root/src/`.
    pub src_root: PathBuf,
    /// Names of the workspace's direct dependencies from which this
    /// package is transitively reachable. Sorted, deduplicated.
    pub root_deps: Vec<String>,
}

/// Run `cargo metadata` in `project_root` and return one `DepPackageInfo`
/// for every non-workspace package in the resolved dependency graph.
///
/// Shells out to `cargo metadata --no-deps=false --format-version 1`.
/// The caller is responsible for caching (Phase 2 concern).
///
/// # Errors
/// Returns `Err` if `cargo` is not found, exits non-zero, or the JSON
/// cannot be parsed.
pub fn discover_dependency_packages(project_root: &Path) -> Result<Vec<DepPackageInfo>>;
```

#### Implementation notes for `discover_dependency_packages`

1. Invoke `cargo_metadata::MetadataCommand::new().current_dir(project_root).exec()?`.

2. Collect workspace member IDs into a `HashSet<PackageId>` for fast
   exclusion.

3. Build a `HashMap<PackageId, Package>` for all packages.

4. Collect direct dependency names of all workspace members:
   - For each workspace member package, iterate `package.dependencies`
     (these are `Dependency` objects with a `name` field).
   - Only consider dependencies whose resolved `PackageId` appears in the
     resolve graph (skip dev-deps that don't appear — filter by checking
     the resolve node's deps list).
   - Simpler alternative accepted: iterate `metadata.resolve.unwrap().nodes`,
     find nodes whose `id` is a workspace member, take their `deps` (which
     are `NodeDep` with `pkg: PackageId`), look up the package name.
     Use this simpler form.

5. BFS for `root_deps`:
   ```
   // root_dep_seeds: HashMap<PackageId, String>
   //   key = package id of each direct dep of any workspace member
   //   value = that package's name (the "root dep" label)
   //
   // reachable: HashMap<PackageId, BTreeSet<String>>
   //   for each non-workspace package, the set of root dep names that reach it
   
   // Initialise: each seed reaches itself under its own name
   for (seed_id, seed_name) in &root_dep_seeds {
       reachable.entry(seed_id).or_default().insert(seed_name.clone());
   }
   
   // Build adjacency: PackageId -> Vec<PackageId> from resolve nodes
   let adjacency: HashMap<PackageId, Vec<PackageId>> = ...;
   
   // Multi-source BFS: for each seed, propagate its name forward
   for (seed_id, seed_name) in &root_dep_seeds {
       let mut queue = VecDeque::from([seed_id.clone()]);
       let mut visited = HashSet::new();
       while let Some(current) = queue.pop_front() {
           if !visited.insert(current.clone()) { continue; }
           for dep_id in adjacency.get(&current).into_iter().flatten() {
               reachable.entry(dep_id.clone()).or_default()
                   .insert(seed_name.clone());
               queue.push_back(dep_id.clone());
           }
       }
   }
   ```
   One BFS per root dep. Total complexity O(root_deps * graph_size) which is
   acceptable (tens of root deps, hundreds of packages).

6. Assemble `Vec<DepPackageInfo>`:
   - For each package in `metadata.packages` where `id` is NOT in
     workspace member IDs:
     - `crate_name = package.name.clone()`
     - `crate_version = package.version.to_string()`
     - `src_root = package.manifest_path.parent().unwrap().into()`
     - `root_deps = reachable.get(&package.id).map(|s| s.iter().cloned().collect()).unwrap_or_default()`
   - Sort output by `crate_name` for deterministic test assertions.

## Testing

All tests are unit tests in `src/deps.rs`, classified as **post-impl**.

**`test_discover_dependency_packages_on_self`** — call
`discover_dependency_packages(Path::new(env!("CARGO_MANIFEST_DIR")))`.
Assertions:
- Returns `Ok(vec)` with at least one entry.
- No entry has `crate_name` equal to `"cargo-debrief"` (workspace members
  excluded).
- At least one known dep (`"anyhow"`, `"serde"`, or `"tokio"`) is present.
- Every entry has non-empty `root_deps` (everything in `Cargo.toml` must be
  reachable from at least one direct dep, which is itself a root dep).
- `src_root` for `"anyhow"` (or any well-known dep) exists on disk
  (`src_root.exists()`).

**`test_root_deps_sorted_deduplicated`** — same call, find `"serde"` in the
result (it's listed as a direct dep in `Cargo.toml`), assert
`root_deps == ["serde"]` (or at least contains `"serde"` and is sorted).

**`test_chunk_origin_serde_round_trip`** (in `src/chunk.rs`) — add a case
to the existing `chunk_bincode_round_trip` test that round-trips a `Chunk`
with `origin: ChunkOrigin::Dependency { .. }`.

## Success Criteria

- `cargo test` passes with no new failures.
- `cargo clippy` passes with no warnings on new code.
- `src/deps.rs` compiles with `cargo check`.
- `discover_dependency_packages` returns entries for at least `anyhow`,
  `serde`, and `tokio` when called on the cargo-debrief workspace.
- Each returned entry has `root_deps` non-empty (for deps that appear in
  `Cargo.toml` directly) or possibly empty (for deeply transitive deps
  not reachable from any workspace member — which should not occur in a
  well-formed workspace).
- `INDEX_VERSION` is `4` in `store.rs`.
- `ChunkOrigin` is the last variant concern on `Chunk` before `embedding`.
