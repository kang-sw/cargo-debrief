# Phase 2: Dependency Indexing Pipeline

## Context

Ticket: `260404-feat-dependency-chunking`

Phase 1 delivered: `ChunkOrigin` on `Chunk`, `DepPackageInfo` struct, and
`discover_dependency_packages()` in `src/deps.rs`. Phase 2 wires this into
an indexing pipeline that produces a separate `deps-index.bin` and adds
staleness detection via `Cargo.lock` hashing.

Phase 3 (unified search + `--no-deps` flag) is explicitly out of scope here.
After Phase 2, the deps index is built and saved to disk but is not yet merged
into search results.

### Key Decisions

**Pipeline location**: New private helpers in `src/service.rs`, not a new
module. The pattern is identical to `run_index` / `ensure_index_fresh`. A
separate `deps_index.rs` module would add indirection for no gain at this
scope; promote to its own module only if Phase 3 or later substantially grows
it.

**`DepsIndexData` storage**: New struct in `src/store.rs` with its own
`DEPS_INDEX_VERSION = 1` constant, serialized to `.git/debrief/deps-index.bin`.
Fully decoupled from `IndexData` and `INDEX_VERSION`. A flat `Vec<Chunk>`
field is used (no per-file keying) because search consumes deps chunks as a
single flat pool. Per-package grouping is a future optimization (Phase 3+).

**`Cargo.lock` hash**: Stored in `DepsIndexData::cargo_lock_hash: Option<String>`.
Implementation: `DefaultHasher` over the raw file bytes, formatted as a 16-digit
hex string. `DefaultHasher` output is deterministic within a process but not
guaranteed across Rust versions. For a staleness check, a hash collision or
spurious invalidation triggers a harmless full dep reindex — acceptable
trade-off that avoids adding a `sha2`/`blake3` dependency.

**Embedder refactor**: `run_index` currently constructs its own `Embedder`
internally. Phase 2 needs to share one `Embedder` across project and dep
indexing in the `index` command. Strategy: refactor `run_index` and
`ensure_index_fresh` to accept `embedder: &Embedder`. All callers
(`InProcessService::index`, `search`, `overview`) construct the embedder
upfront via `make_embedder`. The `overview` method will now construct an
embedder even when the index is fresh — acceptable because `Embedder::load`
skips download when files are cached; the ONNX session load is fast.

**File walking**: Manual recursive `std::fs::read_dir` walk for `.rs` files
under `dep.src_root.join("src")`. No new crate (`walkdir`) needed. The
helper is a small private function in `service.rs`.

**Pub filtering**: After `RustChunker::chunk()`, retain only chunks where
`chunk.metadata.visibility == Visibility::Pub`.

**Embedding text annotation**: Prepend one header line to `embedding_text`
before embedding:
- With root deps: `[dependency] {crate_name} (dependency of: {root_deps joined with ", "})`
- Empty root deps (edge case): `[dependency] {crate_name}`

The original `embedding_text` follows on the next line. `display_text` is
**not** modified — the annotation is for embedding retrieval only.

**Triggering dep indexing**: Added alongside project indexing without a new
trait method for Phase 2. The `index` service method triggers both. Dep
staleness is checked in a new `ensure_deps_index_fresh`, which is called
from `search` alongside `ensure_index_fresh`. `overview` does not load the
dep index (it looks up project file paths only).

## Steps

### Step 1 — Add `DepsIndexData` to `src/store.rs`

```rust
const DEPS_INDEX_VERSION: u32 = 1;

#[derive(Serialize, Deserialize)]
pub struct DepsIndexData {
    version: u32,
    pub cargo_lock_hash: Option<String>,
    pub chunks: Vec<Chunk>,
}

impl DepsIndexData {
    pub fn new() -> Self {
        Self {
            version: DEPS_INDEX_VERSION,
            cargo_lock_hash: None,
            chunks: vec![],
        }
    }
}

pub fn save_deps_index(path: &Path, data: &DepsIndexData) -> Result<()>;
pub fn load_deps_index(path: &Path) -> Result<Option<DepsIndexData>>;
```

`save_deps_index` / `load_deps_index` mirror `save_index` / `load_index`
exactly: create parent dirs on save; return `Ok(None)` on missing file or
version mismatch.

`version` must be the first field (same invariant as `IndexData` — version
mismatch is detected by reading the first 4 bytes).

### Step 2 — Refactor `run_index` and `ensure_index_fresh` in `src/service.rs`

Remove the `let embedder = make_embedder(model_name).await?;` line from
`run_index`. Change its signature to:

```rust
async fn run_index(
    project_root: &Path,
    prior_commit: Option<&str>,
    existing_index: Option<IndexData>,
    embedder: &Embedder,
) -> Result<(IndexData, IndexResult)>
```

Change `ensure_index_fresh` signature to:

```rust
async fn ensure_index_fresh(project_root: &Path, embedder: &Embedder) -> Result<IndexData>
```

Inside `ensure_index_fresh`, pass `embedder` through to `run_index`. The
staleness logic (commit check, model check) is unchanged.

Update all three callers in `InProcessService` to construct the embedder
before calling these helpers:

```rust
// Pattern used in all three methods:
let config = load_config(&config_paths(project_root))?;
let model_name = config.embedding_model.as_deref()
    .unwrap_or(ModelRegistry::DEFAULT_MODEL);
let embedder = make_embedder(model_name).await?;
```

In `search`, the existing second `make_embedder` call (for query embedding)
is removed — reuse the one constructed above.

In `overview`, the embedder is constructed but only consumed inside
`ensure_index_fresh` if the index is stale.

### Step 3 — Add private helpers to `src/service.rs`

**`deps_index_path`** — mirrors `index_path`, returns `.git/debrief/deps-index.bin`:

```rust
fn deps_index_path(project_root: &Path) -> Result<PathBuf>
```

**`cargo_lock_content_hash`** — reads `project_root/Cargo.lock` and returns
a hex hash string:

```rust
fn cargo_lock_content_hash(project_root: &Path) -> Result<String> {
    use std::hash::{Hash, Hasher};
    use std::collections::hash_map::DefaultHasher;
    let bytes = std::fs::read(project_root.join("Cargo.lock"))
        .context("Cargo.lock not found")?;
    let mut hasher = DefaultHasher::new();
    bytes.hash(&mut hasher);
    Ok(format!("{:016x}", hasher.finish()))
}
```

**`collect_dep_rs_files`** — recursively finds `.rs` files under a directory:

```rust
fn collect_dep_rs_files(src_dir: &Path) -> Vec<PathBuf>
```

Returns absolute paths. Does not error on missing `src_dir` — returns empty
vec (some crates have no `src/` in the registry cache).

**`build_dep_embedding_text`** — formats the annotation line:

```rust
fn build_dep_embedding_text(
    crate_name: &str,
    root_deps: &[String],
    original_embedding_text: &str,
) -> String
```

### Step 4 — Add `run_deps_index` to `src/service.rs`

```rust
async fn run_deps_index(
    project_root: &Path,
    embedder: &Embedder,
) -> Result<DepsIndexData>
```

Implementation outline:

1. `let packages = deps::discover_dependency_packages(project_root)?;`
2. `let lock_hash = cargo_lock_content_hash(project_root).ok();` — treat
   missing `Cargo.lock` as `None` (workspace with no lock file is unusual
   but possible).
3. Collect all chunks across all packages:
   ```
   for package in &packages {
       let src_dir = package.src_root.join("src");
       for rs_file in collect_dep_rs_files(&src_dir) {
           let relative = rs_file.strip_prefix(&package.src_root).unwrap_or(&rs_file);
           let source = fs::read_to_string(&rs_file)?;
           let raw_chunks = chunker.chunk(relative, &source)?;
           for chunk in raw_chunks {
               if chunk.metadata.visibility != Visibility::Pub { continue; }
               // annotate embedding_text
               // set chunk.origin = ChunkOrigin::Dependency { ... }
               all_chunks.push(chunk);
           }
       }
   }
   ```
4. Collect `all_embedding_texts`, embed in batches of `EMBED_BATCH_SIZE`
   (same pattern as `run_index`, same progress output format — prefix with
   `indexing deps` instead of `indexing`).
5. Assign embeddings back to chunks by position.
6. Assemble and return:
   ```rust
   DepsIndexData { version: DEPS_INDEX_VERSION, cargo_lock_hash: lock_hash, chunks: all_chunks }
   ```
7. Save to `deps_index_path(project_root)?` via `store::save_deps_index`.

**`ChunkOrigin::Dependency` fields to populate:**
- `crate_name`: `package.crate_name.clone()`
- `crate_version`: `package.crate_version.clone()`
- `root_deps`: `package.root_deps.clone()`

**`chunk.metadata.file_path`**: The `RustChunker` populates this from the
`file_path` argument. Pass `relative` (relative to `src_root`) so the path
is `src/foo.rs` style (matching the chunker's convention). The path stored
in `chunk.metadata.file_path` for dep chunks is package-relative — Phase 3
search display will need to prepend `package.src_root` to reconstruct absolute
paths.

### Step 5 — Add `ensure_deps_index_fresh` to `src/service.rs`

```rust
async fn ensure_deps_index_fresh(
    project_root: &Path,
    embedder: &Embedder,
) -> Result<DepsIndexData>
```

Logic:
1. `let current_hash = cargo_lock_content_hash(project_root).ok();`
2. Load `store::load_deps_index(&deps_index_path(project_root)?)?`
3. If loaded data exists and `data.cargo_lock_hash == current_hash` → return
   data unchanged (fresh).
4. Otherwise → `run_deps_index(project_root, embedder)` and return.

`current_hash == None` (no `Cargo.lock`) always triggers a reindex — correct
behavior, since we cannot determine freshness without the lock file.

### Step 6 — Wire into `InProcessService`

**`InProcessService::index`** — full rebuild of both indexes:

```rust
async fn index(&self, project_root: &Path, _path: &Path) -> Result<IndexResult> {
    let config = ...;
    let embedder = make_embedder(model_name).await?;
    let (_data, result) = run_index(project_root, None, None, &embedder).await?;
    run_deps_index(project_root, &embedder).await?;
    Ok(result)
}
```

The `IndexResult` returned reflects only project chunks (consistent with
current behavior). Dep chunks are a side effect.

**`InProcessService::search`** — add dep index freshness check:

```rust
async fn search(...) -> Result<Vec<SearchResult>> {
    let embedder = make_embedder(model_name).await?;
    let index_data = ensure_index_fresh(project_root, &embedder).await?;
    let _deps_data = ensure_deps_index_fresh(project_root, &embedder).await?;
    // Phase 3 merges _deps_data into search; for now, it is built and saved.
    let search_index = SearchIndex::build(flat_chunks)?;
    search_index.search(query, &embedder, top_k)
}
```

`_deps_data` is unused in Phase 2 (prefixed `_`). The call ensures the dep
index is fresh and serialized to disk. Phase 3 will consume it.

**`InProcessService::overview`** — no change to behavior, but now constructs
embedder upfront to pass to `ensure_index_fresh`. No dep index involvement.

### Step 7 — Import `deps` in `src/service.rs`

Add `use crate::deps;` and `use crate::store::DepsIndexData;` to imports.

## Testing

All tests are **post-impl**. Use `env!("CARGO_MANIFEST_DIR")` as `project_root`
for integration-style tests.

**`test_run_deps_index_on_self`** (in `src/service.rs` tests or integration test):
- Skip in CI if no network (use `#[ignore]` or check for model cache file).
- Calls `run_deps_index` with a pre-loaded `Embedder`.
- Assertions:
  - Returns `Ok(data)` with `data.chunks.len() > 0`.
  - At least one chunk has `origin == ChunkOrigin::Dependency { .. }`.
  - All chunks have `embedding == Some(_)` (not `None`).
  - All chunks have `metadata.visibility == Visibility::Pub` (pub filter held).
  - At least one chunk's `embedding_text` starts with `[dependency]`.

**`test_deps_index_staleness_check`** (unit test, no embedder):
- Construct a `DepsIndexData` with a known `cargo_lock_hash`.
- Write it to a tempdir via `save_deps_index`.
- Load with `load_deps_index` and assert hash is preserved.
- Overwrite first 4 bytes with wrong version; assert `load_deps_index` returns `None`.

**`test_cargo_lock_content_hash_is_deterministic`** (unit test):
- Call `cargo_lock_content_hash` twice on `CARGO_MANIFEST_DIR`.
- Assert both calls return `Ok(same_string)`.
- Assert result is 16 hex characters.

**`test_collect_dep_rs_files_on_self_src`** (unit test):
- Call `collect_dep_rs_files(Path::new(env!("CARGO_MANIFEST_DIR")).join("src"))`.
- Assert result is non-empty and all entries end in `.rs`.

**`test_build_dep_embedding_text`** (unit test):
- `build_dep_embedding_text("serde", &["serde".to_string()], "original text")`
  → assert starts with `[dependency] serde (dependency of: serde)`.
- `build_dep_embedding_text("serde", &[], "original text")` → assert starts
  with `[dependency] serde` and does not contain `(dependency of:`.

## Success Criteria

- `cargo test` passes with no new failures (skip network-dependent tests with
  `#[ignore]` if needed).
- `cargo clippy` passes with no warnings on new or modified code.
- Running `cargo debrief index` on any Rust project produces
  `.git/debrief/deps-index.bin` alongside `.git/debrief/index.bin`.
- The deps index file loads without error via `store::load_deps_index`.
- All chunks in the deps index have `embedding == Some(_)`.
- All chunks in the deps index have `metadata.visibility == Visibility::Pub`.
- All `embedding_text` values in the deps index start with `[dependency]`.
- Running `cargo debrief index` a second time without changing `Cargo.lock`
  skips dep reindexing (the `indexing deps` progress line does not appear).
